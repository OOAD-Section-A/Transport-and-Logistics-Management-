Data Contracts

This subsystem provides a fully pluggable route optimization engine. External teams interact through clearly-defined interfaces only — no coupling to internal implementation details.

Creational
Factory Method
Instantiates the correct optimization strategy at runtime based on IRouteData.getOptimizationMode() from the DB team — without the caller knowing which concrete class is created.
DeliveryRouteStrategyFactory → IRouteStrategy
Structural — Adapter
Router Adapter
Wraps incompatible external Map APIs (Google Maps SDK, OSRM HTTP) behind a single stable IMapRouterPort interface. Swap providers without touching any business logic.
GoogleMapsAdapter / OsrmAdapter → IMapRouterPort
Structural — Facade
Subsystem Facade
Provides a single, simple entry point optimizeDeliveryCycle() that hides the factory, strategy selection, map adapter calls, and observer notifications behind one clean API.
RouteOptimizerFacade (PUBLIC API)
Behavioral — Strategy
Optimization Strategy
Three interchangeable algorithms (Distance, Cost, Priority) share the IRouteStrategy interface. The active algorithm is swapped at runtime — no conditionals scattered through the codebase.
Distance / Cost / Priority → IRouteStrategy
Behavioral — Observer
Route Event Bus
Fleet Tracking and Warehouse subsystems register as listeners. When a route is recalculated, all subscribers are automatically notified via RouteRecalculatedEvent — zero polling required.

Factory Method — Strategy Instantiation
The factory reads IRouteData.getOptimizationMode() and creates the right algorithm object. Adding a new strategy requires zero changes to existing code.
// Abstract creator — defines the template method
abstract class RouteStrategyFactory {
    public final IRouteStrategy getStrategy(IRouteData routeData) {
        return createStrategy(routeData);  // delegates to subclass
    }
    protected abstract IRouteStrategy createStrategy(IRouteData routeData);
}

// Concrete creator — switch on the DB team's mode string
class DeliveryRouteStrategyFactory extends RouteStrategyFactory {
    @Override
    protected IRouteStrategy createStrategy(IRouteData routeData) {
        switch (routeData.getOptimizationMode().toUpperCase()) {
            case "DISTANCE": return new DistanceOptimizationStrategy();
            case "COST":     return new CostOptimizationStrategy();
            case "PRIORITY": return new PriorityOptimizationStrategy();
            default: return new DistanceOptimizationStrategy();
        }
    }
}

// To add a new strategy: implement IRouteStrategy, add one case above. Done.
Adapter Pattern — Map API Isolation
The internal IMapRouterPort interface shields the system from external API changes. Both Google Maps and OSRM produce identical List<RouteSegment> outputs.
// TARGET: stable internal interface my code depends on
interface IMapRouterPort {
    List<RouteSegment> calculateSegments(List<IDeliveryStop> stops);
}

// ADAPTER: translates Google's proprietary SDK → IMapRouterPort
class GoogleMapsAdapter implements IMapRouterPort {
    private final GoogleMapsApiClient client;  // ADAPTEE

    @Override
    public List<RouteSegment> calculateSegments(List<IDeliveryStop> stops) {
        // Calls Google SDK, converts units, returns our RouteSegment objects
        Map<String, Object> resp = client.getDirections(lat1, lng1, lat2, lng2, key);
        double distKm = ((Integer) resp.get("distance_meters")) / 1000.0;
        return List.of(new RouteSegment(fromId, toId, distKm, durMin, cost));
    }
}

// Swap providers: just inject OsrmAdapter — zero business logic changes
RouteOptimizerFacade facade = new RouteOptimizerFacade(
    new OsrmAdapter(new OsrmHttpClient())
);
Strategy Pattern — Runtime Algorithm Swap
All three algorithms share the same interface and are selected at runtime. The facade never has an if/else block for algorithm selection.
interface IRouteStrategy {
    OptimizationResult optimize(List<IDeliveryStop> stops, List<RouteSegment> segments);
    String getStrategyName();
}

// Strategy 1: Nearest Neighbor greedy — minimizes km driven
class DistanceOptimizationStrategy implements IRouteStrategy { ... }

// Strategy 2: Weighted cost matrix — minimizes fuel + tolls - priority bonus
class CostOptimizationStrategy implements IRouteStrategy { ... }

// Strategy 3: SLA-first — highest-priority stops are always served first
class PriorityOptimizationStrategy implements IRouteStrategy { ... }

// Usage in facade — strategy is selected, then called uniformly
IRouteStrategy strategy = strategyFactory.getStrategy(routeData);
OptimizationResult result = strategy.optimize(stops, segments);
Observer Pattern — Automatic Notifications
The event bus decouples this subsystem from Fleet and Warehouse. This subsystem doesn't import their classes — it only calls the interface method.
// CONTRACT: any team that wants notifications implements this
interface IRouteEventListener {
    void onRouteRecalculated(RouteRecalculatedEvent event);
}

// FLEET TEAM'S implementation (lives in their codebase)
class FleetTrackingListener implements IRouteEventListener {
    @Override
    public void onRouteRecalculated(RouteRecalculatedEvent event) {
        updateLiveTruckPositions(event.getVehicleId(), event.getOrderedStopIds());
    }
}

// WAREHOUSE TEAM's implementation (lives in their codebase)
class WarehouseListener implements IRouteEventListener {
    @Override
    public void onRouteRecalculated(RouteRecalculatedEvent event) {
        notifyDispatchDock(event.getDeliveryId());
    }
}

// Registration — called once at application startup
facade.subscribeToRouteEvents(new FleetTrackingListener());
facade.subscribeToRouteEvents(new WarehouseListener());

🗄
Database Team
Implement IRouteData + IDeliveryStop · supply optimization mode string
▼
1

Implement IDeliveryStop

Map your ORM entity (e.g., JPA @Entity) to this interface. Only implement the 6 getter methods.

2

Implement IRouteData

Build this from your delivery record. The critical field is getOptimizationMode() — it must return exactly one of: "DISTANCE", "COST", or "PRIORITY".

3

Call the Facade

Pass your IRouteData to facade.optimizeDeliveryCycle(). You'll receive an OptimizationResult with the ordered stop list, total distance, and cost.

// In the DB team's service layer:
class DeliveryService {
    private final RouteOptimizerFacade routeOptimizer;

    public void triggerRouteOptimization(DeliveryEntity entity) {
        IRouteData data = new DbRouteData(
            entity.getId(),
            entity.getVehicleId(),
            entity.getMode(),    // "DISTANCE" | "COST" | "PRIORITY"
            entity.getDepot(),
            entity.getCapacity(),
            entity.getStops()    // your DbDeliveryStop list
        );
        OptimizationResult result = routeOptimizer.optimizeDeliveryCycle(data);
        // Persist result.orderedStops to your DB if needed
    }
}

Fleet Tracking Team
Implement IRouteEventListener · subscribe at startup
▼
1

Implement IRouteEventListener

Create a class in your codebase that implements this single-method interface. Your implementation decides what to do with each event.

2

Subscribe at Application Startup

Get a reference to the shared RouteOptimizerFacade instance (via dependency injection or a shared service registry) and call subscribeToRouteEvents() once.

// Fleet team's implementation
class FleetTrackingListener implements IRouteEventListener {
    @Override
    public void onRouteRecalculated(RouteRecalculatedEvent event) {
        String vehicleId  = event.getVehicleId();
        List<String> stops = event.getOrderedStopIds();
        double distKm     = event.getTotalDistanceKm();
        long timestamp    = event.getTimestampEpochMs();

        // Your fleet logic here:
        fleetDashboard.updateLiveRouteDisplay(vehicleId, stops);
        gpsTrackerService.setExpectedRoute(vehicleId, stops, distKm);
    }
}

// Register during Spring @PostConstruct or main() init:
routeOptimizerFacade.subscribeToRouteEvents(new FleetTrackingListener());

Warehouse Team
Implement IRouteEventListener · react to dispatch events
▼

Identical integration path as Fleet. Multiple teams can subscribe to the same event — the bus fan-outs to all of them simultaneously.

class WarehouseDispatchListener implements IRouteEventListener {
    @Override
    public void onRouteRecalculated(RouteRecalculatedEvent event) {
        String deliveryId  = event.getDeliveryId();
        String firstStopId = event.getOrderedStopIds().get(0);

        // Trigger dock prep, update loading manifest, alert staff
        dockManagementSystem.prepareDock(deliveryId);
        manifestService.updateLoadingOrder(event.getOrderedStopIds());
    }
}

routeOptimizerFacade.subscribeToRouteEvents(new WarehouseDispatchListener());
